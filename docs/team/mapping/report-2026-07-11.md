## The thrift finding (owner asked to include thrift)
Thrift chains are **not chain-mappable** — same bucket as Ace Hardware:
- Goodwill + Habitat ReStore: every call reached no live human. A diagnostic listen-in on Goodwill
  heard **no menu and no person** = voicemail / recorded info line.
- Every thrift location is independently operated (own phone, own greeting, own hours), so one store
  can't stand in for a chain recipe.
- Did NOT burn calls on Salvation Army / Savers / Unique once the pattern was clear.
- These still work for customers via the **per-store live caller** at check time — there's just no
  chain-wide shortcut to pre-map.

## Small / single-location shops = hit-or-miss (also per-store)
- Independent Card Shop → answered live, locked direct (the one win).
- Comic Book Shop, Cards and Coffee → voicemail, no human.
- Macy's (Toys R Us shop-in-shop) → main line never reached the toy counter; no human.
- PokeMall TCG → not retried this run (single shop; same per-store treatment).
→ Recommendation: leave single-location card/hobby/thrift shops to the live per-store caller (like
  Office Depot / Publix / Ace). The chain-recipe engine is for real multi-store chains.

## Operational note
The local sweep driver kept dying with the container between turns (nohup doesn't survive a reclaim;
local git even reverts to a stale commit — the remote branch is the source of truth). The mapper RUN
itself survives server-side, so the resilient pattern is one-chain-per-check-in driven off mapper/state,
not a long-running local process. Day-2 chains were driven that way.

## Bottom line

per-store, not chain-mappable — no wasted recipes. The biggest remaining value (the 12 locked chains)
is one promote away: ship `6feff66` and they run.
