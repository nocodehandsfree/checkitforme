Admin comps
ADMIN.CHECKITFORME.COM
The admin in the site's skin. Same tokens, same components, same voice. It differs only where operator density demands it. Managed from a phone, so every screen is one thumb wide.
Standing rule
Every new admin feature gets comped on this board first, then built.
Words follow the copy guide. Look follows the style guide. The site board is WEBSITE_COMPS.dc.html in this folder.
Five page types, one grammar
LIVE
Three vitals, then peek rows. The whole business at a glance. 1b
REPORT
Range, hero, wells, one list, footnote. Every intel page uses it. 1g
LOG
Filter pills, day groups, rows with status chips, a Show more key. 1c
CRUD
A list of things you own, a New capsule, an edit sheet. 1e
CONSOLE
Toggle rows and pickers. Settings only, no data. 1i
How this cleans up the mess
Accordions are gone
Every collapsed section becomes a peek row. It shows its headline number and opens its own screen. Nothing hides, nothing stacks.
One report grammar
Store intel, restock intel, kiosk intel, support snapshot, testing summary. All of them read the same: range, hero, wells, one list.
Five vitals become three
Checks today, reach, credits. Money and margin move into the Money peek row. Color reads the state before the number does.
Tooltips become gray lines
Hover does not exist on a phone. Any control that needs explaining gets one gray line under it, written to the copy guide.
Pagers become a key
The floating pager covered list rows on a phone. Long lists end in one raised Show more key instead.
Policy splits in two
Flags and pricing stay as a console. Store requests, waitlist and moderation become queues in the LOG pattern. Zones leaves the admin, the engine stays.
TURN 2 · THE DENSE PAGES
Taming the five heaviest screens
These ship too much at once today. None needs a new pattern — each slots into a page type. The fixes are the same five moves everywhere: one hero answer, wells for the vital few, the status-icon set instead of colored words, every tooltip rewritten as one gray line, and everything past the fold folded into a Show-more key. Nothing is invented — same tokens, same voice.

Launch · a CRUD checklist
The three full-width dropdowns shrink to one scrollable pill row — filters are chrome, not content. The orange wall of restore text becomes one carved strip with a Restore key; the list of names lives one tap away.
Each item: checkbox and title on one line, description clamped to two, phase and tags as small chips, To do muted at the end. Launch-critical reads as a red LAUNCH chip — never a loose colored dot. A progress bar answers "how close" before any row does.

Chains · a long CRUD list
131 rows, so noise is the enemy. The second pill — mapped or unmapped — folds into the meta line as one word. The rating collapses to a single chip: color is the bucket (good, ok, weak) and the number is the grade, so five ratings read on one three-color scale, not a rainbow.
Muted chains dim the whole row so the eye skips them. Unrated ones drop the chip entirely. The list ends in a Show-more key — never a floating pager over the rows.
